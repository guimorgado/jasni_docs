## Overview

Once you've added custom domains to Jasni, you'll need to monitor their health, update configurations, and manage email addresses. This guide covers domain management best practices.

## Viewing Domain Status

Access your domain overview in the [Jasni Dashboard](https://app.jasni.ai):

1. Navigate to **Domains** (under the Mail section in the sidebar)
2. View the list of all configured domains
3. Click on any domain to see detailed status

### Domain Status Indicators

| Status | Description | Action Required |
|--------|-------------|-----------------|
| Fully Verified | All DNS records valid (TXT, MX, SPF, DKIM) | None |
| Partial Setup | Domain verified but some records missing | Add missing DNS records |
| Pending | Domain ownership not yet verified | Add TXT verification record |

Each domain shows individual verification status for:
- **Domain**: TXT verification record
- **MX**: Mail server record (for receiving)
- **SPF**: Sender policy record (for sending)
- **DKIM**: Email signing record (for sending)

## Re-verifying DNS Records

If you've updated your DNS configuration or need to check verification status:

1. Go to **Domains** (under the Mail section in the sidebar)
2. Click on your domain
3. Click **Verify DNS**
4. Jasni will check all DNS records (TXT, MX, SPF, DKIM)

<Note>
  DNS changes can take up to 48 hours to propagate. If verification fails, wait and try again later.
</Note>

## Managing Email Addresses

### Listing Addresses

View all email accounts for your user:

<CodeGroup>
```typescript TypeScript
const { accounts, total } = await jasni.accounts.list()

accounts.forEach(account => {
  console.log(`${account.email} - ${account.name}`)
})
```

```python Python
response = jasni.accounts.list()

for account in response.accounts:
    print(f"{account.email} - {account.name}")
```
</CodeGroup>

### Creating New Addresses

Create a new email account with a custom or auto-generated username:

<CodeGroup>
```typescript TypeScript
// With custom username
const { account } = await jasni.accounts.create({
  username: 'support',
  name: 'Support Team',
})
// Creates: support@jasnimail.com

// With auto-generated username
const { account } = await jasni.accounts.create({
  name: 'My Account',
})
```

```python Python
# With custom username
result = jasni.accounts.create(
    username="support",
    name="Support Team"
)
# Creates: support@jasnimail.com

# With auto-generated username
result = jasni.accounts.create(name="My Account")
```
</CodeGroup>

### Updating Account Display Name

Update the display name of an email account:

<CodeGroup>
```typescript TypeScript
const { account } = await jasni.accounts.update({
  email: 'support@jasnimail.com',
  name: 'Customer Support',
})
```

```python Python
result = jasni.accounts.update(
    "support@jasnimail.com",
    name="Customer Support"
)
```
</CodeGroup>

### Deleting an Account

<CodeGroup>
```typescript TypeScript
await jasni.accounts.delete('oldaccount@jasnimail.com')
```

```python Python
jasni.accounts.delete("oldaccount@jasnimail.com")
```
</CodeGroup>

<Warning>
  Deleting an account will permanently remove all associated emails and data. This action cannot be undone.
</Warning>

## Handling Domain Issues

### Common Issues and Solutions

<AccordionGroup>
  <Accordion title="Sudden increase in bounces">
    1. Check if your DNS records are still valid
    2. Verify you haven't been added to blocklists
    3. Review recent email content for spam triggers
    4. Contact Jasni support if issues persist
  </Accordion>
  <Accordion title="DMARC alignment failures">
    1. Ensure the `From` address matches your domain
    2. Verify SPF includes all sending sources
    3. Check DKIM signatures are valid
    4. Review DMARC reports for specific failures
  </Accordion>
  <Accordion title="Emails delayed or not arriving">
    1. Check domain verification status
    2. Verify MX records if receiving
    3. Review webhook logs for delivery events
    4. Check recipient-side spam folders
  </Accordion>
  <Accordion title="Domain verification expired">
    1. Re-verify DNS records in the dashboard
    2. Ensure no DNS changes were made accidentally
    3. Check for domain transfer or DNS provider changes
  </Accordion>
</AccordionGroup>

## Removing a Domain

If you no longer need a domain:

1. Go to **Domains** (under the Mail section)
2. Click on the domain you want to remove
3. Click the trash icon next to the domain name
4. Confirm the deletion

<Warning>
  Removing a domain will:
  - Disable all email addresses on that domain
  - Stop all incoming email processing
  - Invalidate any webhooks using those addresses
  
  This action cannot be undone.
</Warning>

### Before Removing

- Migrate any active email addresses to a new domain
- Update any integrations using the domain
- Export any necessary data or logs
- Update webhooks to use new addresses
